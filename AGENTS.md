# AGENTS.md

このリポジトリで作業するすべてのAIコーディングエージェント（Claude Code, Codex, Cursor 等）への共通指示。**安全に関わるルールの本体はこのファイルに一本化されている。** ルールを変更するときはこのファイルを編集すること。Claude Code固有の補足はCLAUDE.mdにあるが、ルール自体をそちらへ重複して書かない（二重管理による食い違いを防ぐため）。

## プロジェクト概要

AIを活用したアプリケーション開発用の汎用テンプレート。派生プロジェクトで中身を実装する。ビルド/実行/テストコマンドはCLAUDE.mdに記載される（プロジェクト確定後に埋まる）。

## 安全ルール（全エージェント共通）

1. **破壊的コマンドを実行しない**: `rm` の再帰+強制削除、`git push --force`（`-f`, `--force-with-lease` 含む全変種）、`git reset --hard`、`git clean`。取り消し・回復は「追加する」操作（`git revert`、`git stash`、バックアップブランチ）だけで行う。Claude Codeではこれらは設定により強制ブロックされるが、**他のツールでは強制されないので、このルールを自律的に守ること**。
2. **秘密情報を扱わない**: `.env` 系ファイル・秘密鍵を読まない・コミットしない・出力に含めない。漏えい（コミット履歴への混入等）を見つけたら、履歴の掃除より先にキーの無効化・再発行を案内する。
3. **安全網を弱めない**: `.claude/settings.json` の `permissions.deny`、`.claude/hooks/`、このファイルの安全ルールを黙って削除・緩和しない。緩和を求められた場合は、何が守れなくなるかを平易な言葉で説明してから、依頼されたルール単位でのみ変更する。
4. **テストを添える**: 新しい機能や修正には動作を確認するテストを付ける。書けない変更（設定のみ・見た目のみ等）は、その旨と手動での確認方法を報告に明記する。
5. **壊れた・元に戻したいと言われたら**: `.claude/skills/safe-rollback/SKILL.md` の手順に従う（凍結→切り分け→退避→安全な復旧）。
6. **公開・リリースの前に**: `.claude/skills/go-live-checklist/SKILL.md` の監査を必ず先に実施する。監査を通さずにデプロイ作業へ進まない。
7. **定期点検を頼まれたら**: `.claude/skills/project-health-check/SKILL.md` に従う。
8. **安全網に関わるファイルを変更したら**: `python3 scripts/verify_safety_net.py` を実行して整合性を確認する。

## 手順書（ランブック）の読み替え

ルール5〜7のファイルは、Claude Codeでは自動発動するスキルだが、中身はただのMarkdown手順書であり、どのエージェントでも読んで従える。文中のClaude Code固有機能（`/security-review`、プラグインによるレビュー、AskUserQuestion等）は、使えないツールでは同等の代替（コードを読んでの手動セキュリティレビュー、通常の質問文）で読み替えること。「その機能が無いから手順ごとスキップする」ことはしない。

## 限界

この指示とツール側のブロックは、事故の確率を下げるものであり保証ではない。何が守られていて何が守られていないか（ツールごとの強制力の差を含む）はREADMEの「安全網の範囲と限界」を参照。個人情報・決済等の高リスク用途の公開前には、人間の専門家レビューを推奨する立場を崩さないこと。
